initialize()method
public static function initialize(enterFrameSeed:DisplayObject, versionCheck:String):void

Initializes the timing core for KitchenSync. Must be called before using any actions. By default, the system uses EnterFrameCore as the core but other cores can be specified by calling initializeWithCore() instead.

Parameters

enterFrameSeed:DisplayObject — A display object whose ENTER_FRAME event will trigger updates to the synchronizer. Usually, you can use this in your document class / application root.
 
versionCheck:String (default = NaN) — a string for the version you think you're using. e.g. 1.2 This is recommended but not required. It will throw an error if you're using the wrong version of KS.

See also

initializeWithCore()

Example 
         // In most circumstances you can initialize KS by adding this line to  
         // the main funciton of your program (or in the first frame)
         KitchenSync.initialize(this);
initializeWithCore()method 
public static function initializeWithCore(synchronizerCore:ISynchronizerCore, versionCheck:String):void

Allows you to specify a synchronizer core when you initialize the system. This can be any of the ISynchronizerCores and will allow you to use enterframes, timers, or other mehtods to drive the dispatches of your synchronizer. If this sounds too complicated, you can use the default settings by calling KitchenSync.initialize()

Parameters

synchronizerCore:ISynchronizerCore — The synchronizer core to use to drive updates.
 
versionCheck:String (default = NaN) — a string for the version you think you're using. e.g. 1.2 This is recommended but not required. It will throw an error if you're using the wrong version of KS.
